## Philosophy: Why This Skill Exists

Most AI-generated interfaces suffer from **distributional convergence** — they cluster around the statistical center of design decisions. Inter font. Purple gradient. Card layout. Safe neutrals. Technically correct, visually interchangeable.

**UI/UX Pro Max v5.0 solves this** through:
1. **Curated Design Intelligence** — Embedded data tables that an AI can actually reference (no external databases, no CLI tools)
2. **Bold Creative Direction** — Forced aesthetic commitment before any code is written
3. **Production-Ready Output** — Fully-coded components, animations, and video templates

This skill provides **heuristic-driven design guidance** that an AI assistant can execute during a conversation. It does not require Python scripts, CSV files, or npm packages. All data is embedded directly in this document.

## Module 0: Meta & Principles

### How This Skill Works

This skill is designed for **AI assistants** (Claude, Cursor, etc.) to use during design and code generation. It works through **explicit heuristic instructions** that the AI can execute mentally:

```
CREATIVE BRIEF WORKFLOW (AI-Executable):

Step 1: IDENTIFY
- Read the user's project description
- Extract: industry, audience, platform, tone keywords

Step 2: MATCH (use embedded tables in Module 7)
- Look up industry in Industry-to-Style table → Get recommended styles
- Look up tone keywords in Style table → Select primary style
- Look up style in Palette table → Select palette
- Look up style in Font table → Select pairing

Step 3: COMMIT
- State the chosen style, palette, and fonts explicitly
- Explain WHY they fit the industry/tone
- Declare the ONE memorable differentiator

Step 4: CHECK
- Verify no banned fonts/colors are used
- Verify contrast ratios meet 4.5:1 minimum
- Verify animation respects prefers-reduced-motion
```

### Hard Rules (Never Break)

| # | Rule | Rationale |
|---|------|-----------|
| 1 | **Never output placeholder text** — always use real content or lorem ipsum with context | Professional output |
| 2 | **Never use `!important`** in CSS — use specificity and cascade | Maintainable code |
| 3 | **Never skip focus states** — every interactive element must have visible focus | Accessibility |
| 4 | **Never use px for font sizes** — use rem for accessibility | Responsive typography |
| 5 | **Never hardcode colors** — always use design tokens | Theming support |
| 6 | **Never ignore `prefers-reduced-motion`** — respect user preferences | Accessibility |
| 7 | **Never use `Math.random()` for IDs** — use deterministic generation | Reproducibility |
| 8 | **Never skip mobile-first** — default to mobile, enhance for desktop | Mobile-first |
| 9 | **Never use inline styles for theming** — use CSS variables | Theming |
| 10 | **Never forget loading states** — every async operation needs feedback | UX |
| 11 | **Never default to Inter or Roboto as primary fonts** — these are overused by AI and signal generic output. Always select a distinctive primary font from the curated list. System fonts (`system-ui`, `-apple-system`) are acceptable ONLY as final fallback layers, never as the primary choice. | Distinctive design |
| 12 | **Never use purple-to-pink gradients on white/light backgrounds as the dominant aesthetic** — this is the clichéd "AI slop" default. Gradients are permitted on dark backgrounds or when used as subtle accents, but must not be the dominant visual identity. | Avoid clichés |
| 13 | **Never create timid, evenly-distributed palettes** — use dominant colors with sharp accents (60-30-10 rule) | Bold design |
| 14 | **Never claim "auto-validation" without providing the validation procedure** — always show your work | Credibility |
| 15 | **Never reference external databases or tools that are not embedded in this skill** — if data is required, it must be included | Self-contained |

### Anti-Patterns to Avoid

| Anti-Pattern | Why It's Wrong | Correct Approach |
|---|---|---|
| Hamburger menu on desktop | Hides navigation unnecessarily | Visible nav links on desktop |
| Auto-playing carousels | Disrupts user, accessibility issues | Manual controls or pause on hover |
| Infinite scroll without footer | Users can't reach footer | Pagination or load more button |
| Placeholder text as labels | Confuses users, poor accessibility | Floating labels or separate labels |
| Disabled buttons without explanation | Users don't know why | Show validation errors inline |
| Modal without close button | Traps users | Always provide close mechanism |
| Tiny touch targets (<44px) | Hard to tap on mobile | Minimum 44×44px touch targets |
| Low contrast text | Unreadable for many users | WCAG AA minimum 4.5:1 |
| Scroll hijacking | Breaks user expectations | Native scroll behavior |
| Pop-ups on page load | Annoying, high bounce rate | Trigger based on intent/engagement |

## Module 1: Creative Brief Engine

### The 5-Question Framework

Before generating any design system, answer these 5 questions:

```
┌─────────────────────────────────────────────────────────────────┐
│  CREATIVE BRIEF                                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. PURPOSE                                                     │
│     What problem does this interface solve? Who uses it?        │
│     → Extracts: user persona, primary task, success metric      │
│                                                                 │
│  2. TONE                                                        │
│     Pick an EXTREME aesthetic direction:                        │
│     • Brutally minimal    • Maximalist chaos                    │
│     • Retro-futuristic    • Organic/natural                     │
│     • Luxury/refined      • Playful/toy-like                    │
│     • Editorial/magazine  • Brutalist/raw                       │
│     • Art deco/geometric  • Soft/pastel                         │
│     • Industrial/utilitarian • Cyberpunk/neon                  │
│     • Ethereal/glow       • Data mesh/tech                      │
│     → NEVER converge on common choices (Inter, purple gradient)│
│                                                                 │
│  3. CONSTRAINTS                                                 │
│     • Technical: framework, performance budget, accessibility   │
│     • Business: brand guidelines, legal requirements            │
│     • User: device mix, connectivity, disabilities              │
│                                                                 │
│  4. DIFFERENTIATION                                             │
│     What makes this UNFORGETTABLE?                              │
│     The ONE thing someone will remember after closing the tab.  │
│                                                                 │
│  5. INDUSTRY CONTEXT                                            │
│     What industry? What are competitors doing wrong?            │
│     → Maps to industry rules for strategic alignment           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

**CRITICAL:** Choose a clear conceptual direction and execute with precision. Bold maximalism and refined minimalism both work — the key is intentionality, not intensity.

### Anti-Pattern Detection

The brief engine automatically flags generic AI aesthetics:

| Red Flag | Why It's Wrong | Corrective Action |
|---|---|---|
| Inter / Roboto / Arial / system fonts as primary | Overused by AI, lacks character | Force distinctive font pairing from 36 curated options |
| Purple gradient on white background | Cliched AI default | Demand dominant color + sharp accent from 48 palettes |
| Predictable card layout | Lacks spatial imagination | Push asymmetry, overlap, diagonal flow, grid-breaking |
| Evenly-distributed color palette | Timid, forgettable | Demand 60-30-10 rule: dominant, secondary, accent |
| Scattered micro-interactions | No orchestration | One well-orchestrated page load with staggered reveals |
| Solid color backgrounds | Lacks atmosphere | Gradient meshes, noise textures, geometric patterns |
| Space Grotesk as primary | Overused by AI in 2025-2026 | Ban and substitute with distinctive alternative |

```tsx
// components/Modal.tsx
'use client';
import { useEffect, useRef, useCallback } from 'react';
import { X } from 'lucide-react';
import { cn } from '@/lib/utils';

export interface ModalProps {
  isOpen: boolean;
  onClose: () => void;
  title?: string;
  description?: string;
  children: React.ReactNode;
  size?: 'sm' | 'md' | 'lg' | 'xl' | 'full';
  showClose?: boolean;
}

export function Modal({
  isOpen,
  onClose,
  title,
  description,
  children,
  size = 'md',
  showClose = true,
}: ModalProps) {
  const dialogRef = useRef<HTMLDialogElement>(null);
  const previousFocusRef = useRef<HTMLElement | null>(null);

  const sizeStyles = {
    sm: 'max-w-sm',
    md: 'max-w-md',
    lg: 'max-w-lg',
    xl: 'max-w-xl',
    full: 'max-w-4xl',
  };

  // Handle focus trap and escape
  useEffect(() => {
    if (!isOpen) return;

    previousFocusRef.current = document.activeElement as HTMLElement;
    dialogRef.current?.showModal();

    const dialog = dialogRef.current;
    if (!dialog) return;

    const handleKeyDown = (e: KeyboardEvent) => {
      if (e.key === 'Escape') {
        onClose();
        return;
      }

      if (e.key === 'Tab') {
        const focusableElements = dialog.querySelectorAll<HTMLElement>(
          'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
        );
        const firstElement = focusableElements[0];
        const lastElement = focusableElements[focusableElements.length - 1];

        if (e.shiftKey && document.activeElement === firstElement) {
          e.preventDefault();
          lastElement?.focus();
        } else if (!e.shiftKey && document.activeElement === lastElement) {
          e.preventDefault();
          firstElement?.focus();
        }
      }
    };

    dialog.addEventListener('keydown', handleKeyDown);

    // Focus first focusable element
    const firstFocusable = dialog.querySelector<HTMLElement>(
      'button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])'
    );
    firstFocusable?.focus();

    return () => {
      dialog.removeEventListener('keydown', handleKeyDown);
      previousFocusRef.current?.focus();
    };
  }, [isOpen, onClose]);

  // Handle backdrop click
  const handleBackdropClick = useCallback(
    (e: React.MouseEvent<HTMLDialogElement>) => {
      if (e.target === dialogRef.current) {
        onClose();
      }
    },
    [onClose]
  );

  if (!isOpen) return null;

  return (
    <dialog
      ref={dialogRef}
      onClick={handleBackdropClick}
      className="backdrop:bg-black/50 bg-transparent p-0 m-auto"
      style={{ position: 'fixed', inset: 0, margin: 'auto' }}
      aria-labelledby={title ? 'modal-title' : undefined}
      aria-describedby={description ? 'modal-description' : undefined}
    >
      <div
        className={cn(
          'w-full bg-[var(--color-surface)] rounded-xl shadow-xl',
          'animate-in fade-in zoom-in-95 duration-200',
          sizeStyles[size]
        )}
        role="dialog"
        aria-modal="true"
      >
        {(title || showClose) && (
          <div className="flex items-start justify-between p-6 pb-0">
            <div>
              {title && (
                <h2 id="modal-title" className="text-lg font-semibold text-[var(--color-text)]">
                  {title}
                </h2>
              )}
              {description && (
                <p id="modal-description" className="mt-1 text-sm text-[var(--color-muted)]">
                  {description}
                </p>
              )}
            </div>
            {showClose && (
              <button
                onClick={onClose}
                className="p-2 rounded-lg text-[var(--color-muted)] hover:text-[var(--color-text)] hover:bg-[var(--color-background)] transition-colors cursor-pointer"
                aria-label="Close modal"
              >
                <X size={20} />
              </button>
            )}
          </div>
        )}
        <div className="p-6">{children}</div>
      </div>
    </dialog>
  );
}
```

```tsx
  const context = useContext(ToastContext);
  if (!context) {
    throw new Error('useToast must be used within a ToastProvider');
  }

export function ToastProvider({ children }: { children: React.ReactNode }) {
  const [toasts, setToasts] = useState<Toast[]>([]);

  const addToast = useCallback((toast: Omit<Toast, 'id'>) => {
    const id = Math.random().toString(36).slice(2, 9);
    setToasts((prev) => [...prev, { ...toast, id }]);

    setToasts((prev) => prev.filter((t) => t.id !== id));
```

```tsx
// components/StatCard.tsx
'use client';
import { useEffect, useState } from 'react';
import { cn } from '@/lib/utils';
import { TrendingUp, TrendingDown } from 'lucide-react';

export interface StatCardProps {
  label: string;
  value: number;
  prefix?: string;
  suffix?: string;
  trend?: number;
  trendLabel?: string;
  decimals?: number;
  animate?: boolean;
  sparkline?: number[];
  className?: string;
}

export function StatCard({
  label,
  value,
  prefix = '',
  suffix = '',
  trend,
  trendLabel = 'vs last period',
  decimals = 0,
  animate = true,
  sparkline,
  className,
}: StatCardProps) {
  const [displayValue, setDisplayValue] = useState(animate ? 0 : value);

  useEffect(() => {
    if (!animate) {
      setDisplayValue(value);
      return;
    }

    const duration = 1000;
    const steps = 30;
    const increment = value / steps;
    let current = 0;

    const timer = setInterval(() => {
      current += increment;
      if (current >= value) {
        setDisplayValue(value);
        clearInterval(timer);
      } else {
        setDisplayValue(current);
      }
    }, duration / steps);

    return () => clearInterval(timer);
  }, [value, animate]);

  const formattedValue = displayValue.toLocaleString('en-US', {
    minimumFractionDigits: decimals,
    maximumFractionDigits: decimals,
  });

  return (
    <div className={cn('p-6 bg-[var(--color-surface)] rounded-xl border border-[var(--color-muted)]/20', className)}>
      <p className="text-sm font-medium text-[var(--color-muted)]">{label}</p>
      <div className="mt-2 flex items-baseline gap-2">
        <span className="text-3xl font-bold text-[var(--color-text)]">
          {prefix}{formattedValue}{suffix}
        </span>
        {trend !== undefined && (
          <span
            className={cn(
              'flex items-center text-sm font-medium',
              trend >= 0 ? 'text-[var(--color-success)]' : 'text-[var(--color-error)]'
            )}
          >
            {trend >= 0 ? <TrendingUp className="h-4 w-4 mr-1" aria-hidden="true" /> : <TrendingDown className="h-4 w-4 mr-1" aria-hidden="true" />}
            {Math.abs(trend)}%
          </span>
        )}
      </div>
      {trendLabel && (
        <p className="mt-1 text-xs text-[var(--color-muted)]">{trendLabel}</p>
      )}
      {sparkline && sparkline.length > 0 && (
        <div className="mt-4 h-12" aria-hidden="true">
          <Sparkline data={sparkline} />
        </div>
      )}
    </div>
  );
}

function Sparkline({ data }: { data: number[] }) {
  const min = Math.min(...data);
  const max = Math.max(...data);
  const range = max - min || 1;
  const width = 100;
  const height = 48;

  const points = data
    .map((value, index) => {
      const x = (index / (data.length - 1)) * width;
      const y = height - ((value - min) / range) * height;
      return `${x},${y}`;
    })
    .join(' ');

  return (
    <svg viewBox={`0 0 ${width} ${height}`} className="w-full h-full" aria-hidden="true">
      <polyline
        fill="none"
        stroke="var(--color-primary)"
        strokeWidth="2"
        strokeLinecap="round"
        strokeLinejoin="round"
        points={points}
      />
    </svg>
  );
}
```

### 4.3 Scroll-Triggered Animation Hook

```typescript
// hooks/useScrollAnimation.ts
import { useEffect, useRef, useState } from 'react';

interface UseScrollAnimationOptions {
  threshold?: number;
  rootMargin?: string;
  triggerOnce?: boolean;
}

export function useScrollAnimation(options: UseScrollAnimationOptions = {}) {
  const { threshold = 0.1, rootMargin = '0px', triggerOnce = true } = options;
  const ref = useRef<HTMLDivElement>(null);
  const [isVisible, setIsVisible] = useState(false);

  useEffect(() => {
    const element = ref.current;
    if (!element) return;

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsVisible(true);
          if (triggerOnce) {
            observer.disconnect();
          }
        } else if (!triggerOnce) {
          setIsVisible(false);
        }
      },
      { threshold, rootMargin }
    );

    observer.observe(element);
    return () => observer.disconnect();
  }, [threshold, rootMargin, triggerOnce]);

  return { ref, isVisible };
}

// Usage example
export function AnimatedSection({ children, className }: { children: React.ReactNode; className?: string }) {
  const { ref, isVisible } = useScrollAnimation();

  return (
    <div
      ref={ref}
      className={className}
      style={{
        opacity: isVisible ? 1 : 0,
        transform: isVisible ? 'translateY(0)' : 'translateY(20px)',
        transition: 'opacity 0.5s ease-out, transform 0.5s ease-out',
      }}
    >
      {children}
    </div>
  );
}
```

```tsx
// video/components/AnimatedText.tsx
import { useCurrentFrame, interpolate } from 'remotion';

interface AnimatedTextProps {
  text: string;
  style?: React.CSSProperties;
  startFrame?: number;
  duration?: number;
  type?: 'fade' | 'slide' | 'scale' | 'typewriter';
}

export function AnimatedText({
  text,
  style,
  startFrame = 0,
  duration = 30,
  type = 'fade',
}: AnimatedTextProps) {
  const frame = useCurrentFrame();
  const progress = interpolate(frame, [startFrame, startFrame + duration], [0, 1], {
    extrapolateLeft: 'clamp',
    extrapolateRight: 'clamp',
  });

  const getStyle = (): React.CSSProperties => {
    switch (type) {
      case 'slide':
        return {
          ...style,
          opacity: progress,
          transform: `translateY(${(1 - progress) * 30}px)`,
        };
      case 'scale':
        return {
          ...style,
          opacity: progress,
          transform: `scale(${0.9 + progress * 0.1})`,
        };
      case 'typewriter':
        const charsToShow = Math.floor(progress * text.length);
        return {
          ...style,
        };
      default:
        return { ...style, opacity: progress };
    }
  };

  if (type === 'typewriter') {
    const charsToShow = Math.floor(progress * text.length);
    return <span style={style}>{text.slice(0, charsToShow)}</span>;
  }

  return <span style={getStyle()}>{text}</span>;
}
```

```tsx
    extrapolateRight: 'clamp',
  });

  const getTransform = () => {
    const distance = 100;
    switch (direction) {
      case 'left':
        return `translateX(${(1 - progress) * distance}px)`;
      case 'right':
        return `translateX(${(progress - 1) * distance}px)`;
      case 'up':
        return `translateY(${(1 - progress) * distance}px)`;
      case 'down':
        return `translateY(${(progress - 1) * distance}px)`;
    }
  };

  return (
    <div
      style={{
        transform: getTransform(),
        opacity: progress,
      }}
    >
      {children}
    </div>
  );
}
```

```tsx
      }}
    >
      {logo}
    </div>
  );
```

## Module 6: Quality Assurance

### 6.1 Contrast Validation Procedure

Before delivering any interface, manually verify contrast ratios:

```
### Contrast Check Procedure
For every text/background combination:

1. Get hex codes for foreground (text) and background
2. Convert to RGB: #RRGGBB → (R, G, B) where each component is decimal
3. Calculate relative luminance:
   For each channel (R, G, B):
     - Normalize: X = X/255
     - Apply gamma: Xlin = X <= 0.03928 ? X/1292 : ((X + 0.055)/1.055)^2.4
   - L = 0.2126 * Rlin + 0.7152 * Glin + 0.0722 * Blin

4. Calculate ratio: (L1 + 0.05) / (L2 + 0.05) where L1 is lighter

5. Verify:
   - Normal text (<18px): ratio ≥ 4.5:1 (AA), ≥ 7:1 (AAA)
   - Large text (≥18px bold or ≥24px): ratio ≥ 3:1 (AA), ≥ 4.5:1 (AAA)
   - UI components: ratio ≥ 3:1 (AA)
```
