Combined Prompt: Evaluation & Integration of AI Billing Proxy Repositories

Role: You are an expert in AI tooling, API gateways, billing proxies, open‑source ecosystem analysis, and software architecture. Your task is twofold: (1) evaluate a list of GitHub repositories, then (2) propose three wildly different approaches to enhance, combine, or connect the most promising ones into a target project.

Target Project Definition:
Build a unified, robust billing proxy and API gateway system that supports multiple AI services (opencode, Claude, Gemini, OpenAI, Hermes, Kiro, and similar platforms). The system should handle authentication patching, request relay, usage billing, and rate‑limiting across services, while being maintainable and secure.

---

Phase 1 – Repository Evaluation (based on PromptA)

You are given a list of GitHub repository URLs (below). Ignore any non‑repository URLs (e.g., markdown files, static pages). Evaluate all repositories against the following criteria:

1. Functionality – What does the repo do? (billing proxy, auth fix, API gateway, relay, patch, backdoor, MCP server, etc.)
2. Compatibility – Which AI services or clients does it support? (opencode, Claude, Gemini, OpenAI, Hermes, Kiro, etc.)
3. Billing proxy capability – Does it include or enable a billing proxy mechanism? If yes, describe how.
4. Active maintenance – Last commit date, open issues, pull requests, forks, stars.
5. Code quality & documentation – Clear README, setup instructions, examples.
6. Security & risk – Potential backdoors, unmaintained dependencies, obfuscated code.

Output for Phase 1:

· Select the 15 best repositories (most useful, reliable, and relevant for the target project).
· Present them in a table with these columns:
    Rank | Repo Name | Primary Use | AI Services Supported | Billing Proxy? | Maintenance Status | Key Strength | Risk Level
· Below the table, provide a prognosis for the top‑5 repositories. For each, explain how well it is likely to work with each service (opencode, Claude, Gemini, OpenAI, Hermes, Kiro) and note long‑term issues (detection, rate limiting, legal concerns).

---

Phase 2 – Three Wildly Different Integration Approaches (based on PromptB)

Now, based on your Phase‑1 analysis (especially the top‑15 repositories), propose three distinct strategies to enhance, combine, or connect those repositories (or selected subsets) into the target project (unified billing proxy & API gateway).

For each approach you must:

· Explicitly state which repos are involved and what actions you take:
    upgrade | synergy | merge | amplify | interconnect | integrate
· Provide a structural hierarchy + directional path (text diagram, ASCII art, or nested list) showing components/modules, data/control flow, dependencies, and entry/exit points.
· Explain why this approach is “wildly different” from the other two (conceptually unique – e.g., performance‑focused vs. modularity‑focused vs. business‑logic‑focused).

Before presenting the three approaches, show your step‑by‑step reasoning using these five cognitive skills:

1. Brainstorm – list at least 5‑10 raw ideas (no judgment yet) on how to combine/enhance the repos.
2. Plan – define evaluation criteria (e.g., effort, maintainability, billing accuracy, security) and a high‑level roadmap.
3. Investigate – identify technical risks, integration mismatches (auth methods, API schemas, language differences), and hidden backdoors.
4. Analyze – compare trade‑offs (effort vs. benefit, compatibility) for the top candidate ideas.
5. Examine – validate that each of your final three approaches is truly “wildly different” and aligns with the target project’s goals.

Output Format for Phase 2:

· A concise summary of your step‑by‑step thought process (the five skills applied).
· Then, for each of the three approaches:
  · Name (short, descriptive)
  · Actions taken (list repos + action verbs)
  · Structural hierarchy + path (text diagram)
  · Why this is wildly different
