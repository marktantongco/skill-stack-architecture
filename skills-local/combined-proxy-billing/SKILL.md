# Combined Proxy Billing Evaluation Prompt

## Purpose
Structured evaluation framework for analyzing and integrating AI billing proxy repositories. Provides a two-phase methodology: (1) quantitative repository evaluation against 6 criteria, (2) qualitative integration approach design using 5 cognitive skills.

## Use
Read the prompt file and apply it to any list of billing proxy / API gateway repositories to produce:
1. A ranked top-15 evaluation table
2. A prognosis for the top-5 repositories
3. Three "wildly different" integration approaches with architecture diagrams

## Methodology
Phase 1 Criteria: Functionality, Compatibility, Billing Proxy Capability, Maintenance, Code Quality, Security
Phase 2 Skills: Brainstorm → Plan → Investigate → Analyze → Examine

## Target
Unified billing proxy & API gateway supporting: OpenCode, Claude, Gemini, OpenAI, Hermes, Kiro
